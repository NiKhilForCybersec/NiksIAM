# Conditional Access Policy: Require MFA for All Users
Policy Name: "Require MFA - All Users"
Users: All users
  Exclude: Break-glass accounts (emergency access)
Cloud Apps: All cloud apps
Conditions: All
Grant: Require multifactor authentication

MFA Bypass Attacks

AttackHow It WorksMitigations
MFA Fatigue/Bombing Spam push notifications until user approves Number matching, limit attempts, alerting
AiTM Phishing Proxy captures session token after MFA Phishing-resistant MFA (FIDO2), device compliance
SIM Swapping Attacker takes over phone number Don't use SMS, use authenticator apps
SS7 Interception Intercept SMS via telecom vulnerabilities Don't use SMS, use authenticator apps
Real-time Phishing Attacker relays OTP as user enters it FIDO2 (cryptographically bound to site)

Interview Tips

ğŸ’¡ Key Interview Points
  • Three factors: Know (password), Have (phone/token), Are (biometric)
  • MFA requires: Two factors from DIFFERENT categories
  • Phishing-resistant: FIDO2, Windows Hello - cryptographically bound to legitimate site
  • SMS is weak: SIM swapping, SS7 attacks - avoid if possible
  • MFA fatigue: Counter with number matching, rate limiting
  • Passwordless: Windows Hello, FIDO2 keys, Authenticator passwordless
