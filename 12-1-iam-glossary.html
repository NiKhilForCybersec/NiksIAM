IAM Glossary & Terminology

üéØ Quick Reference

Essential IAM terms and acronyms. Use this as a quick reference during interviews or when reading documentation.

A - C

TermDefinition
ABACAttribute-Based Access Control. Access decisions based on attributes (user department, resource sensitivity, time of day).
Access CertificationPeriodic review where managers confirm users still need their access. Also called attestation or access review.
Access TokenCredential used to access protected resources/APIs. Short-lived (typically 1 hour).
ACLAccess Control List. List of permissions attached to an object specifying who can access it.
Active Directory (AD)Microsoft's on-premises directory service using Kerberos, NTLM, LDAP.
ADFSActive Directory Federation Services. Microsoft's on-prem federation server for SSO.
Authentication (AuthN)Verifying WHO you are. "Prove your identity."
Authorization (AuthZ)Determining WHAT you can do. "Check your permissions."
Azure AD ConnectTool that syncs on-prem AD to Entra ID for hybrid identity.
Birthright AccessAccess automatically granted based on user attributes (department, job title) when they join.
CAEContinuous Access Evaluation. Enables real-time token revocation in Entra ID.
CIAMCustomer Identity and Access Management. IAM for external customers (vs employees).
ClaimsStatements about a user in a token (name, email, groups, roles).
Conditional AccessMicrosoft's Zero Trust policy engine. Evaluates signals, applies access controls.

D - I

TermDefinition
DACDiscretionary Access Control. Owner controls who can access their resource.
Directory ServiceDatabase optimized for storing and querying identity data (AD, LDAP, Entra ID).
EntitlementA specific permission or access right (group membership, role, privilege).
Entra IDMicrosoft's cloud identity service (formerly Azure AD).
FederationTrust relationship between identity providers enabling SSO across organizations.
FIDO2Passwordless authentication standard using hardware security keys. Phishing-resistant.
IAMIdentity and Access Management. Discipline of managing identities and their access.
ID TokenJWT containing user identity claims in OIDC. Used for authentication, NOT API access.
IdPIdentity Provider. System that authenticates users and issues tokens (Okta, Entra ID).
IGAIdentity Governance and Administration. Access certifications, lifecycle management, compliance.

J - O

TermDefinition
JITJust-In-Time. Access granted only when needed, for limited time. Core PAM concept.
JMLJoiner, Mover, Leaver. The three identity lifecycle events.
JWTJSON Web Token. Base64-encoded JSON with header, payload, signature. Used for tokens.
KerberosAuthentication protocol using tickets. Used by Active Directory.
LDAPLightweight Directory Access Protocol. Protocol for querying directories.
Least PrivilegePrinciple of granting minimum access needed to perform a job.
MACMandatory Access Control. System-enforced access based on security labels.
MFAMulti-Factor Authentication. Requires 2+ factors from different categories.
OAuth 2.0Authorization framework for API access. NOT authentication.
OIDCOpenID Connect. Identity layer on OAuth 2.0. Used for authentication.

P - S

TermDefinition
PAMPrivileged Access Management. Securing admin/root access with vaulting, session recording, JIT.
PIMPrivileged Identity Management. Microsoft's JIT solution in Entra ID.
PKCEProof Key for Code Exchange. Secures OAuth for public clients. Pronounced "pixy."
PrincipalEntity that can authenticate (user, service account, application).
ProvisioningCreating accounts and granting access in target systems.
RBACRole-Based Access Control. Access granted through role assignments.
Refresh TokenLong-lived token used to get new access tokens without re-authentication.
SAMLSecurity Assertion Markup Language. XML-based SSO protocol for enterprise apps.
SCIMSystem for Cross-domain Identity Management. Standard for automated user provisioning.
SoDSeparation of Duties. Preventing toxic access combinations (e.g., create + approve).
SPService Provider. Application that relies on IdP for authentication.
SSOSingle Sign-On. One login grants access to multiple applications.

T - Z

TermDefinition
TenantDedicated instance of Entra ID for an organization.
TGTTicket Granting Ticket. Kerberos ticket obtained after initial authentication.
TokenCredential representing identity or authorization (ID token, access token, refresh token).
TrustRelationship where one system accepts identity assertions from another.
UPNUser Principal Name. User identifier in format user@domain.com.
VaultingSecurely storing privileged credentials with check-out/check-in and rotation.
WebAuthnW3C standard for passwordless authentication. Browser API for FIDO2.
Zero TrustSecurity model: "Never trust, always verify." Continuous verification regardless of location.
ZTNAZero Trust Network Access. Secure remote access without traditional VPN.
