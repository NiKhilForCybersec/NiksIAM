# Single view of all access = foundation for governance

ğŸ”§ Security Engineer Day-to-Day Tasks

ğŸ‘¨â€ğŸ’» What You Actually Do

Access Request Management:

  • Review and approve/deny access requests in queue
  • Configure approval workflows for new applications
  • Troubleshoot failed provisioning
  • Handle escalations from business users

Certification Campaigns:

  • Configure and launch quarterly access reviews
  • Monitor campaign completion rates
  • Chase non-responders (escalate to managers' managers)
  • Review and remediate revoked access
  • Generate compliance reports for auditors

Connector Management:

  • Monitor connector health (is data syncing?)
  • Troubleshoot aggregation failures
  • Onboard new applications to IGA
  • Map attributes and entitlements

Role Engineering:

  • Analyze access patterns for role mining
  • Define new business roles with stakeholders
  • Review and update existing roles
  • Configure birthright access rules

Interview Tips

ğŸ’¡ Key Interview Points
  • IGA vs IAM: IGA = governance (who should have access), IAM = operations (authentication)
  • Identity Cube: Aggregated view of one person's access across all systems
  • Key capabilities: Request, Certification, Lifecycle, Roles, SoD, Reporting
  • Compliance: IGA enables SOX, HIPAA, PCI-DSS access control requirements
  • Platforms: SailPoint, Saviynt, One Identity, Oracle
