Microsoft Zero Trust with Entra ID

Zero Trust Conditional Access Policies
# Policy 1: Require MFA for All Users
Users: All users (exclude break-glass)
Apps: All cloud apps
Conditions: All
Grant: Require MFA

# Policy 2: Block Legacy Authentication
Users: All users
Apps: All cloud apps
Conditions: Client apps = Legacy auth clients
Grant: Block

# Policy 3: Require Compliant Device for Sensitive Apps
Users: All users
Apps: Azure Portal, M365 Admin, AWS Console
Conditions: All platforms
Grant: Require compliant device + MFA

# Policy 4: Block High-Risk Sign-ins
Users: All users
Apps: All cloud apps
Conditions: Sign-in risk = High
Grant: Block

# Policy 5: Require Password Change for High-Risk Users
Users: All users
Apps: All cloud apps
Conditions: User risk = High
Grant: Require password change + MFA

# Policy 6: Restrict by Location
Users: All users
Apps: All cloud apps
Conditions: Location = Blocked countries (Russia, North Korea, etc.)
Grant: Block

Zero Trust Architecture Components

ComponentPurposeTechnologies
Identity Provider Central authentication, SSO, MFA Entra ID, Okta, Ping Identity
Policy Engine Real-time access decisions Conditional Access, Okta Policies
Device Trust Verify device health and compliance Intune, Jamf, CrowdStrike, Carbon Black
Network Access Secure access without VPN Zscaler, Cloudflare Access, Palo Alto Prisma
Risk Assessment Real-time risk scoring Entra Identity Protection, Okta ThreatInsight
Microsegmentation Limit lateral movement Illumio, Guardicore, Azure NSGs

Implementation Roadmap

1

Phase 1: Foundation (Months 1-3)

  • Deploy MFA for all users (start with admins)
  • Block legacy authentication
  • Implement SSO for critical apps
  • Enable sign-in and audit logging
2

Phase 2: Device Trust (Months 3-6)

  • Enroll devices in MDM (Intune/Jamf)
  • Define compliance policies
  • Require compliant devices for sensitive apps
  • Implement Hybrid Azure AD Join
3

Phase 3: Risk-Based Access (Months 6-9)

  • Enable Identity Protection
  • Configure risk-based Conditional Access
  • Implement PIM for admin access
  • Deploy ZTNA for remote access
4

Phase 4: Continuous Verification (Months 9-12)

  • Session controls and CAE
  • Behavioral analytics (UEBA)
  • Microsegmentation
  • Continuous monitoring and improvement
