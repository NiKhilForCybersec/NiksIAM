Okta SSO & MFA

Okta provides robust Single Sign-On capabilities with support for SAML, OIDC, and WS-Federation. Combined with flexible MFA options and adaptive authentication, Okta delivers secure yet user-friendly access.

Application Integration

Okta Integration Network (OIN)

The OIN contains 7,000+ pre-built integrations with SSO and provisioning support.

SAML Application Setup

SAML apps use assertion-based authentication where Okta issues signed SAML assertions.

SAML Configuration 
SAML Settings for Service Provider:

Identity Provider Single Sign-On URL:
https://yourcompany.okta.com/app/appname/exk123456/sso/saml

Identity Provider Issuer:
http://www.okta.com/exk123456

X.509 Certificate:
(Download from Okta App > Sign On > SAML Signing Certificates)

Name ID Format: 
urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress

Secure Web Authentication (SWA)

SWA provides SSO for apps that don't support SAML/OIDC using browser plugin password injection.

⚠️ SWA Limitations
  • Requires Okta Browser Plugin
  • Passwords stored in Okta (encrypted)
  • No true federation - password replay
  • Use only for legacy apps without modern auth

Adaptive MFA & ThreatInsight

Okta's adaptive security features adjust authentication requirements based on risk signals.

Behavior Detection +

Okta analyzes user behavior patterns and flags anomalies:

  • New Device: User signing in from unrecognized device
  • New IP: Sign-in from new IP address
  • New City/State/Country: Geographic anomaly
  • Velocity Anomaly: Impossible travel detection
ThreatInsight +

Pre-authentication threat detection:

  • Blocks known malicious IPs before authentication
  • Identifies credential stuffing attacks
  • Leverages Okta's global threat intelligence
  • Actions: Log only, Challenge MFA, or Block

Okta FastPass

Okta FastPass provides passwordless authentication using device-bound credentials.

    OKTA FASTPASS FLOW
    ═══════════════════

    ┌─────────────────┐        ┌─────────────────┐        ┌─────────────────┐
    │    Managed      │        │    Okta         │        │   Protected     │
    │    Device       │        │    Cloud        │        │   Application   │
    │  (Okta Verify)  │        │                 │        │                 │
    └────────┬────────┘        └────────┬────────┘        └────────┬────────┘
             │                          │                          │
             │  1. User opens app       │                          │
             │─────────────────────────────────────────────────────▶
             │                          │                          │
             │                          │  2. Redirect to Okta    │
             │◀─────────────────────────────────────────────────────
             │                          │                          │
             │  3. Okta detects managed device                     │
             │◀─────────────────────────│                          │
             │                          │                          │
             │  4. Push notification to Okta Verify                │
             │◀─────────────────────────│                          │
             │                          │                          │
             │  5. Biometric unlock (Face ID/Touch ID/PIN)        │
             │───────┐                  │                          │
             │       │ Local            │                          │
             │◀──────┘ verification     │                          │
             │                          │                          │
             │  6. Device-bound credential signs challenge         │
             │─────────────────────────▶│                          │
             │                          │                          │
             │                          │  7. Token issued         │
             │                          │─────────────────────────▶│
             │                          │                          │
             │  8. Access granted (NO PASSWORD USED)               │
             │◀─────────────────────────────────────────────────────

    REQUIREMENTS:
    • Okta Verify app installed
    • Device enrolled as managed device
    • Biometric or PIN configured
    • FastPass enabled in authentication policy
