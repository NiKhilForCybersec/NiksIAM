System Log (Your Best Friend)

System Log Queries
# COMMON FILTERS (Admin Console or API)

# Failed logins for a user
actor.alternateId eq "user@company.com" AND outcome.result eq "FAILURE"

# All MFA events
eventType sw "user.mfa"

# Password resets
eventType eq "user.account.reset_password"

# App assignment events
eventType eq "application.user_membership.add"

# Admin actions
actor.type eq "SystemPrincipal" OR eventType sw "system."

# Suspicious: logins from new locations
eventType eq "user.session.start" AND client.geographicalContext.city ne "Expected City"

# API Token usage
eventType eq "system.api_token.use"

# Export via API (for SIEM integration)
GET /api/v1/logs?since=2024-01-01T00:00:00Z&filter=eventType eq "user.session.start"

API Token Best Practices

Security ‚Üí API ‚Üí Tokens ‚Üí Create Token
- Use descriptive name: "SIEM-Integration-Splunk-Prod"
- Document purpose and owner

# TOKEN SECURITY
‚úÖ Treat tokens like passwords (never commit to code)
‚úÖ Store in secrets manager (HashiCorp Vault, AWS Secrets Manager)
‚úÖ Rotate every 90 days
‚úÖ Use scoped tokens when possible (OAuth 2.0 instead of SSWS)
‚úÖ Monitor token usage in System Log

‚ùå Never share tokens between environments (dev/prod)
‚ùå Never embed tokens in source code
‚ùå Don't create tokens with Super Admin (use least privilege)

Interview Tips
