ğŸ¯ What This Page Covers

Privileged accounts are the "keys to the kingdom." This covers what privileged access is, why it's high-risk, and the fundamentals of Privileged Access Management (PAM).

What is Privileged Access?

Privileged access means elevated permissions that allow administrative actions: installing software, changing configurations, accessing all data, creating/deleting accounts.

PAM Best Practices

âœ… Privileged Access Checklist
  • Eliminate standing privileges: No permanent admin rights, use JIT
  • Vault all credentials: No passwords in spreadsheets or heads
  • Rotate credentials: Automatic rotation after each use or on schedule
  • Record sessions: Video/keystroke recording for all privileged access
  • Separate accounts: Admins have regular + admin accounts, never use admin for email
  • MFA everywhere: Especially for privileged access
  • Monitor and alert: Real-time detection of anomalous privileged activity

Interview Tips

ğŸ’¡ Key Interview Points
  • 80% stat: Most breaches involve compromised privileged credentials
  • Four pillars: Vaulting, Session Management, JIT, Secrets Management
  • Tiered model: Tier 0 (DCs), Tier 1 (Servers), Tier 2 (Workstations)
  • Key principle: Higher tier creds never touch lower tier systems
  • JIT: No standing privileges, request access when needed
  • PAM vendors: CyberArk, BeyondTrust, Delinea (formerly Thycotic)
