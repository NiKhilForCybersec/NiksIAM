Common Built-in Roles

RolePermissionsUse Case
Owner Full access + can assign roles to others Subscription admins only
Contributor Create/manage all resources, cannot assign roles DevOps, developers
Reader View all resources, no changes Auditors, monitoring
User Access Admin Manage user access only (assign roles) IAM administrators
Virtual Machine Contributor Manage VMs, not network/storage access VM administrators
Storage Blob Data Reader Read blob data (data plane, not management) Applications reading data

Azure CLI Examples

Azure RBAC CLI Commands
# List role assignments for a resource group
az role assignment list --resource-group Production-RG --output table

# Assign Contributor role to a user on a resource group
az role assignment create \
  --assignee "[email protected]" \
  --role "Contributor" \
  --resource-group "Production-RG"

# Assign role to a managed identity
az role assignment create \
  --assignee-object-id "managed-identity-object-id" \
  --role "Storage Blob Data Reader" \
  --scope "/subscriptions/sub-id/resourceGroups/rg/providers/Microsoft.Storage/storageAccounts/storage1"

# List all built-in roles
az role definition list --query "[].{Name:roleName, Description:description}" --output table

# Remove role assignment
az role assignment delete \
  --assignee "[email protected]" \
  --role "Contributor" \
  --resource-group "Production-RG"

Best Practices

âœ… Azure RBAC Best Practices
  • Least privilege: Use specific roles over Owner/Contributor
  • Groups over users: Assign roles to Entra ID groups, not individuals
  • Managed identities: Use for Azure-to-Azure access, no credentials
  • PIM: Just-in-time for privileged roles (Owner, User Access Admin)
  • Scope appropriately: Assign at lowest necessary scope
  • Review regularly: Use Access Reviews in Entra ID Governance
