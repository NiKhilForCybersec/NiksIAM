On-Premises Key Characteristics
  • Perimeter-based security - Trust internal network, verify at boundary
  • VPN required for remote access
  • AD as single identity source - Kerberos/NTLM authentication
  • Agent-based security - EDR, DLP agents on endpoints
  • Manual governance - Quarterly reviews, spreadsheet tracking

Hybrid Architecture Key Characteristics
  • Dual identity plane - AD + Entra ID synchronized via Entra Connect
  • Conditional Access extends everywhere - Including on-prem apps via App Proxy

  • Single DLP policy - Purview covers endpoints, cloud apps, and email

Zero Trust Key Principles
  • Verify explicitly - Authenticate and authorize based on all available data points
  • Least privileged access - JIT/JEA, risk-based adaptive policies
  • Assume breach - Minimize blast radius, segment access, verify end-to-end encryption
  • Identity is the perimeter - No trusted network, every request verified
  • Continuous validation - Not just at login, throughout the session
