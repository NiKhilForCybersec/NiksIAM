SCIM Provisioning

SCIM (System for Cross-domain Identity Management) is the standard protocol for automated user provisioning.

OperationSCIM ActionWhat Happens
CreatePOST /UsersNew user created in target app
UpdatePATCH /Users/{id}Attributes synced to target app
DeactivatePATCH active=falseUser disabled in target app
DeleteDELETE /Users/{id}User removed from target app
Group PushPOST /GroupsGroup synced to target app

  • Create attribute mappings for new HR fields

Troubleshooting Provisioning

Common Issues & Resolution
# ISSUE: User not provisioned to app
CHECK:
1. Is user assigned to the app? (directly or via group)
2. Does user match provisioning scope rules?
3. Is provisioning enabled for the app?
4. Check System Log â†’ filter by user + app

# ISSUE: SCIM errors in System Log
eventType eq "application.provision.user.create" AND outcome.result eq "FAILURE"

COMMON ERRORS:

â€¢ 500 Server Error â†’ App-side issue, contact app vendor

# ISSUE: Attributes not syncing
CHECK:
1. Attribute mapping configured correctly?
2. Source attribute populated in Okta profile?
3. Profile Master settings (which source wins?)
4. Force sync: Applications â†’ App â†’ Provisioning â†’ Force Sync

# ISSUE: Deprovisioning not working
CHECK:
1. Deprovisioning enabled in app settings?
2. Deactivate vs Delete setting?
3. App supports SCIM deprovisioning?
4. Manual deprovisioning required for some apps
