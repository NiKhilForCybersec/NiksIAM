âš ï¸ Common Mistake: "OAuth is for Login"

OAuth 2.0 is authorization, not authentication. OpenID Connect (OIDC) adds authentication on top of OAuth. Use OIDC when you need to know WHO the user is.

ID Token vs Access Token

JWT Token Comparison
// ID TOKEN (Authentication) - WHO you are
{
  "sub": "user-id-12345",
  "name": "Sarah Chen",
  "email": "schen@company.com",
  "aud": "client-app-id",      // For the CLIENT
  "auth_time": 1699900000,
  "amr": ["pwd", "mfa"]        // How they authenticated
}

// ACCESS TOKEN (Authorization) - WHAT you can do
{
  "sub": "user-id-12345",
  "aud": "api://backend",       // For the API
  "scp": "Files.Read Files.Write",  // Permissions
  "roles": ["User", "Contributor"],
  "groups": ["engineering-staff"]
}

Troubleshooting: 401 vs 403

ErrorMeaningRoot CauseFix
401 Unauthorized AuthN Failed Token missing, expired, or invalid Re-authenticate, check token
403 Forbidden AuthZ Failed User identified but lacks permission Check groups, roles, policies
