AD to Entra ID Migration Guide

Migrating from on-premises Active Directory to cloud-native Entra ID requires careful planning. This guide covers migration strategies, prerequisites, and step-by-step procedures.

Migration Strategies

    MIGRATION STRATEGY OPTIONS
    ═══════════════════════════

    OPTION 1: HYBRID (Long-term coexistence)
    ─────────────────────────────────────────
    ┌──────────────┐         ┌──────────────┐
    │     AD       │◀───────▶│   Entra ID   │
    │ (On-Premises)│  Sync   │   (Cloud)    │
    └──────────────┘         └──────────────┘
    
    • Keep AD as source of truth
    • Sync users/groups to cloud
    • Supports legacy apps requiring AD
    • Indefinite coexistence


    OPTION 2: CLOUD-FIRST (New deployments)
    ────────────────────────────────────────
    ┌──────────────┐
    │   Entra ID   │  ← Single source of truth
    │   (Cloud)    │
    └──────────────┘
    
    • New organizations without AD
    • Greenfield deployments
    • No on-premises dependencies


    OPTION 3: FULL MIGRATION (AD decommission)
    ──────────────────────────────────────────
    ┌──────────────┐         ┌──────────────┐         ┌──────────────┐
    │     AD       │ ──────▶ │   Hybrid     │ ──────▶ │  Cloud-Only  │
    │   (Start)    │         │   (Interim)  │         │   (Target)   │
    └──────────────┘         └──────────────┘         └──────────────┘
    
    Phase 1: Establish hybrid     Phase 2: Migrate workloads    Phase 3: Decommission AD
    
    • Multi-year journey for most organizations
    • Requires app modernization
    • Must eliminate all AD dependencies

Assessment & Planning

Discovery Checklist

Area Assessment Items Migration Impact
Users & Groups Total users, group nesting depth, distribution lists Sync configuration, license requirements
Applications Kerberos apps, NTLM apps, LDAP binds, GPO-dependent apps App modernization, App Proxy needs
Devices Domain-joined devices, server count, OS versions Device re-join strategy, GPO to Intune
Network DC locations, VPN dependencies, branch offices Connectivity requirements, latency
Security Service accounts, privileged groups, GPOs Managed identities, PIM, Intune policies
PowerShell - AD Assessment Script 
# AD Migration Assessment Script

# User counts
$userCount = (Get-ADUser -Filter *).Count
$enabledUsers = (Get-ADUser -Filter {Enabled -eq $true}).Count
$guestAccounts = (Get-ADUser -Filter {Description -like "*guest*"}).Count

# Group analysis
$allGroups = Get-ADGroup -Filter *
$securityGroups = ($allGroups | Where-Object { $_.GroupCategory -eq 'Security' }).Count
$distributionGroups = ($allGroups | Where-Object { $_.GroupCategory -eq 'Distribution' }).Count

# Nested groups (potential sync issues)
$nestedGroups = foreach ($group in $allGroups) {
    $members = Get-ADGroupMember -Identity $group -ErrorAction SilentlyContinue
    $nestedCount = ($members | Where-Object { $_.objectClass -eq 'group' }).Count
    if ($nestedCount -gt 0) {
        [PSCustomObject]@{
            GroupName = $group.Name
            NestedGroups = $nestedCount
        }
    }
}

# Service accounts
$serviceAccounts = Get-ADUser -Filter {ServicePrincipalName -like "*"} -Properties ServicePrincipalName

# Computers by OS
$computers = Get-ADComputer -Filter * -Properties OperatingSystem | 
    Group-Object OperatingSystem | 
    Select-Object Name, Count

# GPO count
$gpoCount = (Get-GPO -All).Count

# Output summary
Write-Host "=== AD Migration Assessment ===" -ForegroundColor Cyan
Write-Host "Total Users: $userCount (Enabled: $enabledUsers)"
Write-Host "Security Groups: $securityGroups"
Write-Host "Distribution Groups: $distributionGroups"
Write-Host "Groups with Nesting: $($nestedGroups.Count)"
Write-Host "Service Accounts (SPN): $($serviceAccounts.Count)"
Write-Host "GPO Count: $gpoCount"
Write-Host ""
Write-Host "Computers by OS:"
$computers | Format-Table -AutoSize

Prerequisites

Licensing Requirements +
  • Entra ID P1: Hybrid identity, Conditional Access, Self-Service Password Reset with writeback
  • Entra ID P2: Identity Protection, PIM (recommended for migration)
  • Intune: Device management, configuration profiles (GPO replacement)
  • Microsoft 365: E3/E5 for productivity workloads
Technical Prerequisites +
  • AD forest functional level 2003 or higher
  • UPN suffix matches verified custom domain
  • No duplicate proxyAddresses or UPNs
  • Outbound HTTPS (443) connectivity for sync server
  • Server for Entra Connect (Windows Server 2016+)

Staged Migration Approach

    STAGED MIGRATION PHASES
    ════════════════════════

    PHASE 1: FOUNDATION (Weeks 1-4)
    ────────────────────────────────
    ┌─────────────────────────────────────────────────────────────────┐
    │  • Deploy Entra Connect with Password Hash Sync                  │
    │  • Enable Seamless SSO                                          │
    │  • Configure password writeback                                  │
    │  • Deploy pilot Conditional Access policies                      │
    │  • Enable Security Defaults or baseline CA policies              │
    └─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
    PHASE 2: AUTHENTICATION (Weeks 5-12)
    ─────────────────────────────────────
    ┌─────────────────────────────────────────────────────────────────┐
    │  • Roll out MFA to all users                                     │
    │  • Deploy passwordless authentication (FIDO2, WHfB)             │
    │  • Migrate from ADFS to cloud authentication (if applicable)    │
    │  • Enable Self-Service Password Reset                           │
    └─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
    PHASE 3: APPLICATIONS (Weeks 13-26)
    ────────────────────────────────────
    ┌─────────────────────────────────────────────────────────────────┐
    │  • Inventory all applications using AD authentication           │
    │  • Migrate SaaS apps to Entra ID SSO                            │
    │  • Deploy Application Proxy for on-premises apps                │
    │  • Migrate Kerberos apps to cloud auth where possible           │
    │  • Enable user provisioning (SCIM) for key apps                 │
    └─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
    PHASE 4: DEVICES (Weeks 27-40)
    ───────────────────────────────
    ┌─────────────────────────────────────────────────────────────────┐
    │  • Deploy Hybrid Entra ID Join for existing devices              │
    │  • Configure Intune enrollment                                  │
    │  • Migrate GPO settings to Intune configuration profiles        │
    │  • Begin Entra ID Join for new devices                          │
    │  • Pilot cloud-only device management                           │
    └─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
    PHASE 5: GOVERNANCE (Weeks 41-52)
    ─────────────────────────────────
    ┌─────────────────────────────────────────────────────────────────┐
    │  • Implement PIM for privileged roles                            │
    │  • Configure Access Reviews                                      │
    │  • Deploy Entitlement Management                                │
    │  • Enable Identity Protection policies                          │
    │  • Plan AD decommission (if applicable)                         │
    └─────────────────────────────────────────────────────────────────┘

Application Migration

App Type Current Auth Migration Target Complexity
SaaS (Gallery) Various Entra ID SSO (SAML/OIDC) Low
Custom Web (Modern) ADFS/WS-Fed Entra ID SAML/OIDC Medium
On-Prem Web (IIS) Windows Auth/Kerberos Application Proxy + KCD Medium
Legacy (Thick Client) NTLM/Kerberos May require AD (long-term) High
Service Accounts AD service accounts Managed Identities, Workload ID Medium-High

Cutover Planning

⚠️ Before AD Decommission
  • All devices Entra ID Joined (not Hybrid Joined)
  • All applications migrated to cloud authentication
  • No Kerberos/NTLM dependencies remaining
  • All GPOs replaced with Intune policies
  • File servers migrated to SharePoint/OneDrive or Azure Files
  • Print servers migrated to Universal Print
  • All service accounts converted to managed identities
