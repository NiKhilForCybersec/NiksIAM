Seamless SSO (Azure AD)

Seamless SSO Flow
# HOW SEAMLESS SSO WORKS
1. User logged into Windows domain (has Kerberos TGT)
2. User accesses cloud app
3. Cloud app redirects to Entra ID
4. Entra ID sends hidden request to on-prem endpoint
5. AD issues Kerberos ticket for AZUREADSSOACC$ computer
6. Browser sends ticket to Entra ID
7. Entra ID validates ticket, issues tokens
8. User logged in - NO PASSWORD PROMPT!
