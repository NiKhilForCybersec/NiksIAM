ğŸ¯ What This Page Covers

Microsoft Entra ID (formerly Azure AD) is Microsoft's cloud identity platform. This covers tenant structure, licensing, core features, and how it integrates with the Microsoft ecosystem.

What is Entra ID?

Entra ID is Microsoft's cloud-based identity and access management service. It's the identity backbone for Microsoft 365, Azure, and thousands of SaaS applications.

ğŸ“ Name Change

Azure Active Directory was renamed to Microsoft Entra ID in July 2023. Same product, new name. Azure AD and Entra ID are used interchangeably.

Entra ID vs Active Directory

FeatureActive DirectoryEntra ID
Deployment On-premises servers (DCs) Cloud service (Microsoft-hosted)
Protocols Kerberos, NTLM, LDAP SAML, OIDC, OAuth 2.0, WS-Fed
Structure Hierarchical (OUs, forests, domains) Flat (tenants, no OUs)
Policy Group Policy (GPOs) Conditional Access, Intune
Device Join Domain Join Azure AD Join, Hybrid Join
Best For On-prem resources, legacy apps Cloud apps, SaaS, modern auth

Core Features

One login for all applications. Entra ID supports:

  • SAML: Enterprise apps (Salesforce, ServiceNow)
  • OIDC/OAuth: Modern apps, APIs
  • Password-based: Legacy apps (form fill)
  • Linked: Apps with own IdP

Gallery has 5000+ pre-integrated apps, or add custom.

Zero Trust policy engine. Evaluates every sign-in:

  • Signals: User, device, location, app, risk
  • Decisions: Allow, block, require MFA, require compliant device
  • Enforcement: Real-time access control

Requires P1 license.

Just-in-time privileged access:

  • Eligible: User can activate role when needed
  • Time-bound: Access expires automatically
  • Approval: Optional approval workflow
  • Audit: Full trail of activations

Covers Entra roles, Azure RBAC, and group membership. Requires P2.

ML-based risk detection:

  • User risk: Leaked credentials, unusual behavior
  • Sign-in risk: Anonymous IP, impossible travel, malware-linked IP
  • Remediation: Block, require MFA, require password change

Requires P2.
