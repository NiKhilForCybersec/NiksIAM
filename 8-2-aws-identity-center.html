Permission Sets

Permission sets define what users can do in AWS accounts. They create IAM roles in each assigned account.

Permission Set Examples
# AWS MANAGED PERMISSION SETS
AdministratorAccess    # Full admin
PowerUserAccess        # Full access except IAM
ReadOnlyAccess         # View only
ViewOnlyAccess         # Even more restricted view
DatabaseAdministrator  # RDS, DynamoDB admin
NetworkAdministrator   # VPC, Route53, CloudFront

# CUSTOM PERMISSION SET (JSON Policy)
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:ListBucket",
        "ec2:Describe*"
      ],
      "Resource": "*"
    },
    {
      "Effect": "Deny",
      "Action": [
        "ec2:TerminateInstances",
        "s3:DeleteBucket"
      ],
      "Resource": "*"
    }
  ]
}

CLI/SDK Access

AWS CLI with Identity Center
# Configure SSO profile
aws configure sso
# Enter: SSO start URL, region, account, role

# Login via browser
aws sso login --profile my-sso-profile

# Use the profile
aws s3 ls --profile my-sso-profile

# Or set default
export AWS_PROFILE=my-sso-profile

# ~/.aws/config entry created:
[profile my-sso-profile]
sso_start_url = https://d-1234567890.awsapps.com/start
sso_region = us-east-1
sso_account_id = 123456789012
sso_role_name = PowerUserAccess

Interview Tips

ğŸ’¡ Key Interview Points
  • Renamed: AWS SSO is now AWS Identity Center (2022)
  • Permission Sets: Define permissions, assigned to users/groups per account
  • Creates IAM Roles: Permission sets create roles in each target account
  • Federation: SAML 2.0 with external IdP (Okta, Entra) is common
  • SCIM: Automatic user/group provisioning from IdP
  • Free: Identity Center itself is free
