Okta Lifecycle Management

Okta Lifecycle Management automates user provisioning, deprovisioning, and profile synchronization across applications. This reduces manual work and ensures timely access changes.

SCIM Provisioning

SCIM (System for Cross-domain Identity Management) is the standard protocol for automated provisioning.

Provisioning Features by App

Feature Description OIN Support
Create Users Create accounts when assigned Common
Update User Attributes Sync profile changes Common
Deactivate Users Disable when unassigned Common
Sync Password Push password to app Limited
Push Groups Create groups in target app Some apps
Import Users Import existing app users Some apps
API - Trigger Provisioning 
# Assign user to application (triggers provisioning)
curl -X PUT "https://${OKTA_DOMAIN}/api/v1/apps/${APP_ID}/users/${USER_ID}" \
  -H "Authorization: SSWS ${API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "'"${USER_ID}"'",
    "scope": "USER",
    "profile": {
      "salesforceRole": "Standard User",
      "salesforceProfile": "Chatter Free User"
    }
  }'

# Remove user from application (triggers deprovisioning)
curl -X DELETE "https://${OKTA_DOMAIN}/api/v1/apps/${APP_ID}/users/${USER_ID}" \
  -H "Authorization: SSWS ${API_TOKEN}"

# Force sync for all assigned users
curl -X POST "https://${OKTA_DOMAIN}/api/v1/apps/${APP_ID}/lifecycle/activate" \
  -H "Authorization: SSWS ${API_TOKEN}"

Profile Mapping

Profile mapping controls how attributes flow between Okta and applications.

Profile Mapping Expressions 
// Direct mapping
user.firstName

// Concatenation
user.firstName + " " + user.lastName

// Conditional (null check)
user.department != null ? user.department : "Unknown"

// String manipulation
String.toUpperCase(user.countryCode)
String.substring(user.email, 0, String.indexOf(user.email, "@"))

// Array operations  
Arrays.contains(user.groups, "Admins")
Arrays.flatten(user.groups)

// Date formatting
Time.format(user.startDate, "yyyy-MM-dd")

// Complex conditional
user.employeeType == "Contractor" ? "External" : 
  (user.department == "IT" ? "IT Staff" : "Employee")
