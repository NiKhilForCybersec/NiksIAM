Authentication Issues

AADSTS50076: MFA Required but not completed +

Cause: Conditional Access policy requires MFA but user hasn't completed it.

Resolution:

  • Ensure user has registered MFA methods
  • Check if user is excluded from MFA policies incorrectly
  • Verify the app supports MFA claims
  • For legacy apps, consider App Passwords or modern auth migration
AADSTS50105: User not assigned to application +

Cause: Enterprise app has "Assignment required" enabled but user isn't assigned.

Resolution:

  • Assign user directly or via group to the enterprise app
  • Or disable "Assignment required" in app properties (not recommended for sensitive apps)
PowerShell - Check Assignment
# Check if user is assigned to app
Get-MgServicePrincipalAppRoleAssignedTo -ServicePrincipalId $spId | 
    Where-Object { $_.PrincipalId -eq $userId }
AADSTS53003: Access blocked by Conditional Access +

Cause: A Conditional Access policy is blocking access.

Resolution:

  • Check Sign-in logs for "Conditional Access" tab details
  • Identify which policy is blocking (policy name shown in logs)
  • Use "What If" tool to simulate and identify blocking policy
  • Verify user meets policy requirements (device compliance, location, etc.)
AADSTS50126: Invalid username or password +

Cause: Credentials are incorrect or account is locked.

Resolution:

  • Verify username is correct (check UPN vs sAMAccountName)
  • Check if account is locked in AD or Entra ID
  • For synced users, verify password hash sync is working
  • Reset password if necessary

SAML Debugging

Browser - Capture SAML Response 
Steps to capture SAML assertion:

1. Open browser Developer Tools (F12)
2. Go to Network tab
3. Enable "Preserve log" 
4. Attempt SSO sign-in
5. Look for POST to the ACS URL
6. Find SAMLResponse in Form Data
7. Decode Base64 to view XML assertion

Online tools: samltool.com, base64decode.org

Common SAML issues:
• Entity ID mismatch between IdP and SP
• ACS URL incorrect or HTTP vs HTTPS
• Certificate expired or wrong cert configured
• Clock skew (NotBefore/NotOnOrAfter)
• NameID format mismatch

Entra Connect Sync Issues

PowerShell - Entra Connect Diagnostics 
# Check sync status
Get-ADSyncScheduler

# Force full sync
Start-ADSyncSyncCycle -PolicyType Initial

# Force delta sync
Start-ADSyncSyncCycle -PolicyType Delta

# Check for sync errors
Get-ADSyncConnectorRunStatus

# View export errors
$connector = Get-ADSyncConnector -Name "yourtenant.onmicrosoft.com - AAD"
Get-ADSyncCSObject -ConnectorName $connector.Name | 
    Where-Object { $_.ErrorDescription }

# Check password hash sync status
Get-ADSyncAADPasswordSyncConfiguration -SourceConnector "yourdomain.com"

# Common sync issues:
# - UPN suffix not verified in Entra ID
# - Duplicate proxyAddresses
# - Invalid characters in attributes
# - Object filtered by OU or attribute filter

Provisioning Errors

User not provisioned to application +

Checklist:

  • Is user assigned to the app (directly or via group)?
  • Is provisioning enabled and in "On" status?
  • Check provisioning logs for errors
  • Verify attribute mappings are correct
  • Check scoping filters aren't excluding user
  • For SCIM apps, verify API credentials are valid
Provisioning quarantined +

Cause: Too many errors (>60% failure rate).

Resolution:

  • Check provisioning logs for error details
  • Common causes: invalid API credentials, rate limiting, schema changes
  • Fix underlying issue, then restart provisioning
  • May need to clear quarantine and restart sync

KQL - Troubleshooting Queries 
// Find all sign-in failures for a user
SigninLogs
| where UserPrincipalName == "user@domain.com"
| where ResultType != "0"
| project TimeGenerated, AppDisplayName, ResultType, ResultDescription,
    ConditionalAccessStatus, IPAddress, DeviceDetail
| order by TimeGenerated desc

// Conditional Access policy failures
SigninLogs
| where ConditionalAccessStatus == "failure"
| extend CADetails = parse_json(ConditionalAccessPolicies)
| mv-expand CADetails
| where CADetails.result == "failure"
| project TimeGenerated, UserPrincipalName, AppDisplayName,
    PolicyName = CADetails.displayName,
    FailureReason = CADetails.failureReason

// Provisioning failures
AADProvisioningLogs
| where ResultType == "Failure"
| project TimeGenerated, ServicePrincipal, TargetSystem,
    Action, ResultDescription, ModifiedProperties
