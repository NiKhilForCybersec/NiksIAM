Access Certification

Access certification campaigns ensure users have appropriate access through periodic reviews.

Campaign Types

  • User Access Review: Manager reviews all access for their direct reports
  • Resource Access Review: App owner reviews all users with access
  • Entitlement Review: Review specific high-risk entitlements

Access Requests

Self-service access request workflows enable users to request access with appropriate approvals.

Access Request Configuration 
Access Request Policy Example:

Resource: Salesforce - Admin Profile
─────────────────────────────────────

Request Settings:
  ☑ Users can request this resource
  ☑ Require business justification
  
Approval Workflow:
  Stage 1: Manager Approval (required)
  Stage 2: App Owner Approval (required)
  
Time Constraints:
  ☑ Time-bound access
  Default Duration: 90 days
  Maximum Duration: 365 days
  
Auto-Revocation:
  ☑ Auto-revoke when access expires
  ☑ Send reminder 7 days before expiration

Compliance Reporting

Report Type Description Compliance Use
User Access Report All applications and groups for a user Auditor requests, termination verification
Application Access Report All users with access to an application SOX, periodic access review
Certification History Past certification decisions Audit evidence
System Log All authentication and admin events Security audit, incident investigation
