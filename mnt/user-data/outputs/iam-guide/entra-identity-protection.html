Identity Protection

Microsoft Entra ID Protection uses machine learning to detect and remediate identity-based risks. It provides automated risk detection, investigation, and remediation capabilities.

Risk Detection Types

User Risk Detections

Detection Description Detection Type
Leaked credentials User credentials found in dark web or data breaches Offline
Anomalous user activity Behavior patterns inconsistent with user's history Offline
User reported suspicious activity User reports via "This wasn't me" in MFA prompt Real-time
Possible attempt to access PRT Primary Refresh Token access attempt detected Offline

Sign-in Risk Detections

Detection Description Detection Type
Anonymous IP address Sign-in from anonymous proxy (Tor, VPN) Real-time
Atypical travel Sign-in from location atypical for user Offline
Impossible travel Sign-ins from distant locations in impossible time Offline
Malware-linked IP Sign-in from IP known to host malware Offline
Unfamiliar sign-in properties Properties not seen recently for user Real-time
Password spray Multiple accounts targeted with same password Offline
Token issuer anomaly Abnormal SAML token issuer detected Offline

Risk-Based Policies

Terraform - Risk Policies via Conditional Access 
# User Risk Policy - Require password change for high-risk users
resource "azuread_conditional_access_policy" "user_risk_high" {
  display_name = "CA-Risk: Require password change for high-risk users"
  state        = "enabled"

  conditions {
    users {
      include_users = ["All"]
      exclude_users = var.break_glass_users
    }
    applications {
      include_applications = ["All"]
    }
    user_risk_levels = ["high"]
  }

  grant_controls {
    operator          = "AND"
    built_in_controls = ["mfa", "passwordChange"]
  }
}

# Sign-in Risk Policy - Require MFA for medium/high risk sign-ins
resource "azuread_conditional_access_policy" "signin_risk" {
  display_name = "CA-Risk: Require MFA for risky sign-ins"
  state        = "enabled"

  conditions {
    users {
      include_users = ["All"]
      exclude_users = var.break_glass_users
    }
    applications {
      include_applications = ["All"]
    }
    sign_in_risk_levels = ["high", "medium"]
  }

  grant_controls {
    operator          = "OR"
    built_in_controls = ["mfa"]
  }
}

Remediation Actions

PowerShell - Risk Remediation 
# Get risky users
Get-MgRiskyUser -Filter "riskLevel eq 'high'" | 
    Select-Object UserPrincipalName, RiskLevel, RiskLastUpdatedDateTime

# Dismiss user risk (confirmed safe)
Invoke-MgDismissRiskyUser -UserIds @("user-object-id")

# Confirm user compromised (triggers remediation)
Invoke-MgConfirmRiskyUserCompromised -UserIds @("user-object-id")

# Revoke all refresh tokens for user
Revoke-MgUserSignInSession -UserId "user@domain.com"

# Force password reset at next sign-in
Update-MgUser -UserId "user@domain.com" -PasswordProfile @{
    ForceChangePasswordNextSignIn = $true
}
