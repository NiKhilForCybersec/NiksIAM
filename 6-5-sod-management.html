Separation of Duties (SoD)

ğŸ¯ What This Page Covers

SoD prevents fraud by ensuring no single person has conflicting access. This covers toxic combinations, SoD policies, violation detection, and remediation.

What is SoD?

Separation of Duties ensures that critical tasks require multiple people, preventing fraud and errors. No single person should be able to both create AND approve transactions.

Common Toxic Combinations

ProcessConflicting AccessRisk
Procure-to-PayCreate Vendor + Approve PaymentGhost vendor fraud
Order-to-CashCreate Customer + Issue CreditFictitious credits
HR/PayrollCreate Employee + Process PayrollGhost employee fraud
IT AdminCreate User + Approve AccessUnauthorized access
Financial ClosePost Journal + Approve JournalFraudulent entries
InventoryReceive Goods + Adjust InventoryTheft concealment

# When user requests access that would create conflict:
# Option 1: BLOCK - Deny the request
# Option 2: ALERT - Allow but notify risk team
# Option 3: MITIGATE - Allow with compensating control

# Compensating Controls:
# - Additional approval required for transactions
# - Enhanced monitoring/audit logging
# - Dollar amount limits
# - Time-bound access

Interview Tips

ğŸ’¡ Key Interview Points
  • SoD purpose: Prevent fraud by requiring multiple people for critical tasks
  • Toxic combination: Two access rights that together enable fraud
  • Common conflict: Create + Approve in any financial process
  • Compensating controls: When SoD can't be enforced (small team), add monitoring
  • SOX requirement: SoD controls required for financial systems
