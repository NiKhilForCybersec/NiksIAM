Hybrid Identity

Hybrid identity connects on-premises Active Directory with Microsoft Entra ID, enabling users to access both cloud and on-premises resources with a single identity.

Hybrid Authentication Methods

    AUTHENTICATION METHOD COMPARISON
    ═════════════════════════════════

    1. PASSWORD HASH SYNCHRONIZATION (PHS)
    ───────────────────────────────────────
    ┌────────┐         ┌──────────────┐         ┌──────────┐
    │  User  │────────▶│   Entra ID   │         │    AD    │
    │        │  Auth   │              │  Hash   │          │
    │        │◀────────│  Validates   │◀────────│  (sync)  │
    └────────┘         │  hash        │         └──────────┘
                       └──────────────┘
    
    ✓ Simplest to deploy       ✓ Cloud resilience
    ✓ No on-prem dependency    ✓ Leaked credential detection
    
    ─────────────────────────────────────────────────────────────────────

    2. PASS-THROUGH AUTHENTICATION (PTA)
    ─────────────────────────────────────
    ┌────────┐         ┌──────────────┐         ┌──────────┐
    │  User  │────────▶│   Entra ID   │────────▶│    AD    │
    │        │  Auth   │              │ Validate│          │
    │        │◀────────│   (proxy)    │◀────────│(on-prem) │
    └────────┘         └──────────────┘         └──────────┘
                              │
                              ▼
                       ┌──────────────┐
                       │  PTA Agent   │
                       │ (on-premises)│
                       └──────────────┘
    
    ✓ Password never leaves on-prem    ✓ Honors AD policies
    ✗ Requires on-prem agents          ✗ Depends on connectivity
    
    ─────────────────────────────────────────────────────────────────────

    3. FEDERATION (ADFS)
    ─────────────────────
    ┌────────┐         ┌──────────────┐         ┌──────────┐
    │  User  │────────▶│   Entra ID   │────────▶│   ADFS   │
    │        │         │   (trusts    │  SAML   │          │
    │        │◀────────│    ADFS)     │◀────────│(on-prem) │
    └────────┘         └──────────────┘         └──────────┘
                                                      │
                                                      ▼
                                                ┌──────────┐
                                                │    AD    │
                                                └──────────┘
    
    ✓ Full control over auth       ✓ Smart card/cert auth
    ✓ Third-party MFA              ✗ Complex infrastructure
    ✗ Single point of failure      ✗ Higher maintenance
✓ Microsoft Recommendation

Microsoft recommends Password Hash Synchronization (PHS) as the primary authentication method for most organizations due to its simplicity, resilience, and security features like leaked credential detection.

Microsoft Entra Connect

Prerequisites

  • Windows Server 2016 or later (not a Domain Controller)
  • SQL Server Express (included) or SQL Server
  • .NET Framework 4.7.2 or later
  • Entra ID Global Administrator account
  • Enterprise Administrator in AD
  • Outbound connectivity to Microsoft endpoints

Installation

PowerShell - Entra Connect Prerequisites 
# Verify server is NOT a Domain Controller
$dcInfo = Get-WmiObject -Class Win32_ComputerSystem
if ($dcInfo.DomainRole -ge 4) {
    Write-Error "Cannot install on Domain Controller"
}

# Check .NET version
$netVersion = (Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full").Release
if ($netVersion -lt 528040) {
    Write-Warning "Install .NET Framework 4.8"
}

# Verify TLS 1.2 is enabled
$tlsSettings = Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocols\TLS 1.2\Client" -ErrorAction SilentlyContinue
if (-not $tlsSettings.Enabled) {
    Write-Warning "Enable TLS 1.2"
}

# Test connectivity to required endpoints
$endpoints = @(
    "login.microsoftonline.com",
    "secure.aadcdn.microsoftonline-p.com",
    "*.msappproxy.net"
)

foreach ($endpoint in $endpoints) {
    Test-NetConnection -ComputerName $endpoint -Port 443
}

PowerShell - Manage Entra Connect 
# Import Entra Connect module
Import-Module ADSync

# Check sync status
Get-ADSyncScheduler

# Start manual sync (delta)
Start-ADSyncSyncCycle -PolicyType Delta

# Start manual sync (full)
Start-ADSyncSyncCycle -PolicyType Initial

# View sync errors
Get-ADSyncConnectorRunStatus

# Get connector space statistics
Get-ADSyncConnectorStatistics -ConnectorName "contoso.com"

# View custom sync rules
Get-ADSyncRule | Where-Object {$_.Name -like "Custom*"}

# Export sync configuration
Get-ADSyncServerConfiguration -Path "C:\SyncConfig"

Password Writeback

Password writeback enables SSPR and password changes in Entra ID to sync back to on-premises AD.

Requirements

  • Entra ID P1 or P2 license
  • Entra Connect with password writeback enabled
  • AD functional level: Windows Server 2008 R2 or later
  • Service account with password reset permissions
PowerShell - Enable Password Writeback 
# Verify password writeback is enabled
Get-ADSyncAADPasswordResetConfiguration

# Check connector account permissions
$connector = Get-ADSyncConnector -Name "contoso.com"
$account = $connector.ConnectivityParameters.Parameter | 
    Where-Object {$_.Name -eq "forest-login-user"} | 
    Select-Object -ExpandProperty Value

# Grant required permissions to connector account
# Run on Domain Controller
$ou = "OU=Users,DC=contoso,DC=com"
$acl = Get-Acl "AD:\$ou"

# Reset Password permission
$ace = New-Object System.DirectoryServices.ActiveDirectoryAccessRule(
    [System.Security.Principal.NTAccount]"CONTOSO\AADConnect",
    "ExtendedRight",
    "Allow",
    [Guid]"00299570-246d-11d0-a768-00aa006e0529"  # Reset Password
)
$acl.AddAccessRule($ace)
Set-Acl "AD:\$ou" $acl
⚠️ Password Writeback Security
  • The Entra Connect service account can reset ANY user's password in synced OUs
  • Consider excluding privileged accounts from sync
  • Monitor password reset events in AD audit logs
  • Enable account lockout writeback for security
