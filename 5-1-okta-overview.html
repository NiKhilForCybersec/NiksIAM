Okta is a leading cloud Identity Provider (IdP). This covers architecture, admin console navigation, common configurations, integration patterns, and day-to-day operations for IAM engineers.

What is Okta?

Okta is a cloud-native Identity-as-a-Service (IDaaS) platform providing SSO, MFA, lifecycle management, and API access management. It serves as the central identity hub connecting users to all their applications.

Common User Operations

Okta API: User Management
# Create user
POST https://company.okta.com/api/v1/users?activate=true
{
  "profile": {
    "firstName": "Sarah",
    "lastName": "Chen",
    "email": "[email protected]",
    "login": "[email protected]",
    "department": "Engineering"
  },
  "credentials": {
    "password": { "value": "TempP@ss123!" }
  }
}

# Get user
GET https://company.okta.com/api/v1/users/[email protected]

# Update user profile
POST https://company.okta.com/api/v1/users/{userId}
{
  "profile": {
    "department": "Product"
  }
}

# Suspend user (temporary disable)
POST https://company.okta.com/api/v1/users/{userId}/lifecycle/suspend

# Unsuspend user
POST https://company.okta.com/api/v1/users/{userId}/lifecycle/unsuspend

# Deactivate user (offboarding)
POST https://company.okta.com/api/v1/users/{userId}/lifecycle/deactivate

# Delete user (permanent)
DELETE https://company.okta.com/api/v1/users/{userId}

# Reset password
POST https://company.okta.com/api/v1/users/{userId}/lifecycle/reset_password

# List user's app assignments
GET https://company.okta.com/api/v1/apps?filter=user.id eq "{userId}"

# List user's groups
GET https://company.okta.com/api/v1/users/{userId}/groups

Troubleshooting

Check System Log:

Reports â†’ System Log â†’ Filter by user

Look for:
- user.authentication.auth_via_* - Shows auth attempts
- user.session.start - Successful login
- policy.evaluate_sign_on - Which policy evaluated

Common issues:
- MFA required but not enrolled
- Sign-on policy blocking (location, device)
- Account locked or suspended
- Password expired
- AD authentication failure (if delegated auth)

Check SAML assertion:

  • Use browser SAML tracer extension
  • Verify Audience (Entity ID) matches
  • Verify ACS URL is correct
  • Check certificate not expired
  • Verify NameID format matches app expectation

Check app assignment:

  • Is user assigned to app (directly or via group)?
  • Is provisioning complete (if SCIM)?
