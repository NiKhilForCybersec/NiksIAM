Access Certifications

ğŸ¯ What This Page Covers

Access certifications (attestations/access reviews) are periodic reviews where managers confirm users still need their access. Required for SOX, HIPAA, PCI-DSS compliance.

Certification Types

TypeReviewerScopeFrequency
Manager CertificationUser's managerAll access for direct reportsQuarterly
Application CertificationApp/data ownerAll users with app accessAnnually
Entitlement CertificationRole/group ownerUsers with specific entitlementQuarterly
Event-DrivenNew managerInherited team accessOn job change
Privileged AccessSecurity teamAdmin accountsMonthly

Reviewer Decisions

DecisionActionWhen to Use
Approve/CertifyAccess retainedUser still needs access for job
Revoke/RemoveAccess removedUser no longer needs access
DelegateSend to another reviewerReviewer doesn't know user/system
No Decision (timeout)Configurable defaultReviewer didn't respond

Best Practices

âœ… Certification Best Practices
  • Right-size scope: Don't overwhelm reviewers with 500+ items
  • Provide context: Show last login date, role descriptions
  • Default to revoke: If no response, safer to remove access
  • Automate remediation: Auto-revoke after certification closes
  • Track completion: Escalate non-responders
  • Micro-certifications: Smaller, more frequent is better than annual big-bang

Interview Tips

ğŸ’¡ Key Interview Points
  • Compliance: SOX, HIPAA, PCI-DSS all require periodic access reviews
  • Types: Manager (by user), Application (by app), Entitlement (by permission)
  • Key metric: Completion rate, time to complete, revocation rate
  • Rubber-stamping: High approve rates may indicate reviewers not actually reviewing
  • Platforms: SailPoint, Saviynt, Entra ID Governance
