Okta Fundamentals

Okta is a leading cloud-native Identity-as-a-Service (IDaaS) platform providing SSO, MFA, lifecycle management, and API access management. This guide covers Okta's architecture, core concepts, and administrative fundamentals.

API - User Management 
# Create user via Okta API
curl -X POST "https://${OKTA_DOMAIN}/api/v1/users?activate=true" \
  -H "Authorization: SSWS ${API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "profile": {
      "firstName": "Alice",
      "lastName": "Smith",
      "email": "alice.smith@example.com",
      "login": "alice.smith@example.com",
      "mobilePhone": "+1-555-123-4567",
      "department": "Engineering",
      "employeeNumber": "EMP001"
    },
    "credentials": {
      "password": { "value": "TempP@ssw0rd!" }
    }
  }'

# Get user by login
curl -X GET "https://${OKTA_DOMAIN}/api/v1/users/alice.smith@example.com" \
  -H "Authorization: SSWS ${API_TOKEN}"

# Update user profile
curl -X POST "https://${OKTA_DOMAIN}/api/v1/users/${USER_ID}" \
  -H "Authorization: SSWS ${API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "profile": {
      "department": "Sales"
    }
  }'

# Deactivate user
curl -X POST "https://${OKTA_DOMAIN}/api/v1/users/${USER_ID}/lifecycle/deactivate" \
  -H "Authorization: SSWS ${API_TOKEN}"

Okta Groups

Okta-Managed Groups

Groups created and managed directly in Okta Universal Directory.

  • Manual membership management
  • Can be used for app assignments, policies, and provisioning
  • Supports nested groups (limited)

Pushed Groups (AD/LDAP)

Groups synced from Active Directory or LDAP directories.

  • Membership managed in source directory
  • Read-only in Okta
  • Can be used like Okta groups for assignments
  • Requires AD/LDAP agent

Group Rules (Dynamic Groups)

Automatically manage group membership based on user attributes.

Okta Expression Language - Group Rules 
# All users in Engineering department
user.department == "Engineering"

# Users in US or Canada
user.countryCode == "US" OR user.countryCode == "CA"

# Users with specific manager
user.managerId == "00u1234567890"

# Users matching email domain
String.stringContains(user.email, "@contractors.example.com")

# Full-time employees only
user.employeeType == "FTE" AND user.status == "ACTIVE"

# Complex condition
(user.department == "Sales" OR user.department == "Marketing") 
  AND user.countryCode == "US" 
  AND user.userType != "Contractor"

Okta APIs & Automation

API Authentication Methods

  • SSWS Token: API token created in Admin Console (simple, long-lived)
  • OAuth 2.0: Service app with client credentials or private key JWT
  • Session Token: For user-context operations
Python - Okta SDK Example 
from okta.client import Client as OktaClient
import asyncio

# Initialize client
config = {
    'orgUrl': 'https://yourcompany.okta.com',
    'token': 'YOUR_API_TOKEN'
}
okta_client = OktaClient(config)

async def main():
    # List all users
    users, resp, err = await okta_client.list_users()
    for user in users:
        print(f"{user.profile.login} - {user.status}")
    
    # Create a group
    group_profile = {
        'name': 'Engineering Team',
        'description': 'All engineering staff'
    }
    group, resp, err = await okta_client.create_group({'profile': group_profile})
    print(f"Created group: {group.id}")
    
    # Add user to group
    await okta_client.add_user_to_group(group.id, user_id)
    
    # Assign app to group
    app_group_assignment = {
        'id': group.id,
        'priority': 0
    }
    await okta_client.create_application_group_assignment(app_id, group.id, app_group_assignment)

asyncio.run(main())
💡 Okta Rate Limits

Okta enforces rate limits per API endpoint. Common limits:

  • /users: 600 requests/minute
  • /groups: 600 requests/minute
  • /apps: 600 requests/minute
  • Concurrent requests: Varies by endpoint

Always implement exponential backoff and check response headers for rate limit info.
