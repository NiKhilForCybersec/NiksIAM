Custom SAML Integration

SAML Configuration
# WHEN: App not in OIN, but supports SAML

Applications ‚Üí Create App Integration ‚Üí SAML 2.0

# STEP 1: GENERAL SETTINGS
App name: "Internal HR Portal"
App logo: Upload
App visibility: Show in Okta dashboard

# STEP 2: SAML SETTINGS (Get from app vendor)
Single Sign-On URL:    https://hrportal.company.com/saml/acs
Audience URI:          https://hrportal.company.com
Name ID format:        EmailAddress
Name ID value:         user.email

# STEP 3: ATTRIBUTE STATEMENTS
firstName  ‚Üí user.firstName
lastName   ‚Üí user.lastName
department ‚Üí user.department
employeeId ‚Üí user.employeeNumber

# STEP 4: DOWNLOAD METADATA
Download Okta metadata XML ‚Üí Upload to app
OR manually configure:

‚îú‚îÄ‚îÄ IdP Issuer: http://www.okta.com/xxx
‚îî‚îÄ‚îÄ X.509 Certificate: Download and upload to app

SWA (Secure Web Authentication)

SWA for Legacy Apps
# WHEN: App doesn't support SAML/OIDC (legacy, no SSO capability)
# HOW: Okta stores credentials and auto-fills login form

Applications ‚Üí Create App Integration ‚Üí SWA

# CONFIGURATION:
Login URL: https://legacyapp.company.com/login
Username field: #username (CSS selector)
Password field: #password (CSS selector)
Login button: #submit

# CREDENTIAL OPTIONS:
1. User sets own password (user manages)
2. Admin sets password (shared account)
3. External password sync (from another system)

# LIMITATIONS:
‚Ä¢ Requires Okta browser plugin
‚Ä¢ Passwords stored in Okta (not true SSO)
‚Ä¢ No automatic provisioning
‚Ä¢ Security: Okta knows user's app password

# BEST PRACTICE: Use SWA only as last resort
  Plan to migrate app to SAML/OIDC

üîß Security Engineer Day-to-Day Tasks

üë®‚Äçüíª What You Actually Do

App Onboarding:

  • Evaluate new app - check if in OIN first
  • Gather SAML/OIDC requirements from vendor
  • Configure SSO integration
  • Test with pilot users
  • Configure provisioning if supported
  • Assign to appropriate groups

Troubleshooting:

  • "SSO not working" ‚Üí Check SAML tracer, verify config matches
  • "User not provisioned" ‚Üí Check assignment, provisioning logs
  • "Wrong attributes" ‚Üí Review attribute mapping
  • "SWA not filling" ‚Üí Verify selectors, check plugin installed

Maintenance:

  • Update apps when vendor changes endpoints
  • Rotate certificates before expiry
  • Review and update attribute mappings
  • Monitor app usage and remove unused apps

Interview Tips

üí° Key Interview Points
  • OIN: 7,000+ pre-built integrations, check here first
  • SAML vs OIDC: SAML for enterprise, OIDC for modern/APIs
  • SWA: Last resort for legacy apps, not true SSO
  • Provisioning: Many OIN apps support SCIM
  • Attributes: Map user data from Okta to app requirements
